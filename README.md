# Друк етикеток НП (Code 128)

Кросплатформний десктоп-додаток (Windows + macOS) для друку етикеток зі штрих-кодом **Code 128** на термопринтері **Xprinter XP-365B** (203 dpi, direct thermal).

Сценарій: вводите номер ТТН Нової Пошти → додаток перевіряє накладну через API НП → якщо накладна існує, друкує етикетку зі штрих-кодом через системний діалог друку. Превʼю на екрані — це той самий PNG, що йде на друк (WYSIWYG).

## Стек

- Electron + TypeScript (strict), electron-vite, React
- bwip-js — генерація Code 128
- electron-builder — інсталятори NSIS `.exe` і `.dmg`
- safeStorage (Keychain на macOS / DPAPI на Windows) — шифрування API-ключа

## Розробка

```bash
npm install
npm run dev        # запуск у режимі розробки
npm run typecheck  # перевірка типів
npm run lint       # ESLint
```

## Збірка інсталяторів

```bash
npm run build:win  # Windows: NSIS .exe (збирати на Windows)
npm run build:mac  # macOS: два .dmg — Apple Silicon (arm64) і Intel (x64)
```

Артефакти зʼявляються в `dist/` (для macOS — окремі файли `…-arm64.dmg` і `…-x64.dmg`).

### Підпис коду (не реалізовано, лише нотатка)

- **macOS:** без підпису Gatekeeper блокує запуск. Для дистрибуції потрібен сертифікат **Apple Developer ID Application** + нотаризація (`notarize: true` в `electron-builder.yml` і змінні `APPLE_ID`/`APPLE_APP_SPECIFIC_PASSWORD`/`APPLE_TEAM_ID`). Локально неподписаний додаток можна відкрити через правий клік → «Open» або `xattr -dr com.apple.quarantine "/Applications/NP Label Printer.app"`.
- **Windows:** без підпису SmartScreen показує попередження «Невідомий видавець». Потрібен сертифікат code signing (OV/EV), параметри `win.certificateFile`/`certificatePassword` або `signtool` у CI.

## Налаштування

1. Відкрийте вкладку **Налаштування**.
2. Вставте **API-ключ Нової Пошти** (кабінет НП → Налаштування → Безпека → API-ключі) і натисніть «Зберегти». Ключ шифрується через safeStorage і зберігається в `userData`; у відкритому вигляді ніде не лежить.
3. «Перевірити ключ» виконує тестовий запит до НП.
4. За потреби змініть розмір етикетки (за замовчуванням **60×40 мм**) — він використовується і для превʼю, і для розміру сторінки друку.

## Принтер Xprinter XP-365B

- **Windows:** встановіть офіційний драйвер Xprinter (XP-365B), у властивостях принтера задайте розмір носія 60×40 мм.
- **macOS:** принтер працює через CUPS; встановіть драйвер Xprinter для Mac (або сумісний CUPS-драйвер). Додайте принтер у System Settings → Printers & Scanners. Драйвер примхливий: якщо друк не йде, перевірте чергу CUPS на `http://localhost:631`.
- У системному діалозі друку виберіть XP-365B і розмір паперу, що відповідає етикетці.

## Ручне тестування на реальному принтері

- [ ] Валідна ТТН → «Накладну знайдено», превʼю зі штрих-кодом, статус і місто отримувача відповідають накладній.
- [ ] Вигадана ТТН (14 цифр) → «Накладну не знайдено», кнопка друку недоступна.
- [ ] Без API-ключа → банер із підказкою про Налаштування, друк заблоковано.
- [ ] «Друкувати» відкриває системний діалог; розмір сторінки = розміру етикетки з налаштувань.
- [ ] Надрукована етикетка збігається з превʼю; штрих-код зчитується сканером.
- [ ] Скасування діалогу друку не підвішує додаток; повторний друк працює.
- [ ] Зміна розміру етикетки в Налаштуваннях змінює і превʼю, і сторінку друку.
- [ ] Після перезапуску додатка ключ і розмір етикетки збережені.
