# Промпт для Claude Code — десктоп-додаток друку етикеток (Нова Пошта → Code 128)

## Роль і ціль
Збудуй кросплатформний десктоп-додаток (**Windows + macOS**) для друку етикеток зі штрих-кодом **Code 128** на термопринтері **Xprinter XP-365B** (203 dpi, direct thermal).

Сценарій: користувач вводить номер ТТН Нової Пошти → додаток перевіряє існування накладної через API НП → якщо накладна існує, дозволяє надрукувати етикетку зі штрих-кодом Code 128 (номер ТТН) через **системний діалог друку**.

## Стек (фіксований — не замінювати)
- Scaffold: **electron-vite**, шаблон `react-ts`
- **Electron** (остання стабільна) + **TypeScript** (strict) скрізь: main, preload, renderer
- **React + Vite** (renderer)
- **bwip-js** — генерація Code 128
- **electron-builder** — пакетування `.exe` (NSIS) + `.dmg`
- Зберігання секретів: Electron **safeStorage** (Keychain на macOS / DPAPI на Windows)

UI — мінімальний і чистий, без важких UI-фреймворків (достатньо звичайного CSS). Пріоритет — коректність і надійність, не візуальний лиск.

## Структура проєкту (electron-vite layout)
```
src/
  main/        # main process: вікно, IPC-хендлери, друк
  preload/     # contextBridge API
  renderer/    # React
    components/  # TtnInput, LabelPreview, Settings
    lib/         # novaposhta.ts, label.ts
    App.tsx
shared/        # спільні типи, константи (LABEL, назви IPC-каналів, типи відповіді НП)
```
`LABEL` (розмір етикетки в мм) — **єдине джерело правди** у `shared/`, використовується і в CSS `@page`, і в `pageSize` друку, щоб вони ніколи не розʼїхались. Дефолт: `{ width: 60, height: 40 }`.

## Безпека Electron (обовʼязково)
- `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`
- Renderer не має прямого доступу до Node — лише через preload + `contextBridge`
- Усі вхідні дані IPC валідувати в main
- API-ключ НП **ніколи** не тримати в renderer / localStorage. Лише в main через `safeStorage`. Renderer отримує тільки факт «ключ заданий / ні» та може записати новий ключ через IPC.

## Екран 1 — Головний
- Поле вводу номера ТТН (лише цифри, ~14 знаків; trim + видалення пробілів)
- Debounce ~500 мс → перевірка в НП
- Стани: `idle / loading / valid / not-found / error / no-api-key`
- Якщо ключ НП не заданий → підказати перейти в Налаштування (друк заблоковано)
- Якщо накладна існує → показати `LabelPreview` + активна кнопка **«Друкувати»**
- Якщо накладної немає (`StatusCode === "0000"`) → «Накладну не знайдено», друк заблоковано
- За наявності показати дані з відповіді (статус, місто отримувача) для підтвердження, що це потрібна накладна

## Екран 2 — Налаштування
- Поле вводу API-ключа НП
- «Зберегти» → IPC → `safeStorage.encryptString` → запис у файл у `app.getPath('userData')`
- Індикатор «ключ збережено» (показувати лише факт/маску, не сам ключ)
- «Перевірити ключ» — тестовий запит до НП
- (Опційно) поля розміру етикетки (ширина×висота, мм) з дефолтом 60×40, зберігати в конфіг

## Nova Poshta — контракт інтеграції (`lib/novaposhta.ts`)
**Endpoint:** `POST https://api.novaposhta.ua/v2.0/json/`

**Body:**
```json
{
  "apiKey": "<key>",
  "modelName": "TrackingDocument",
  "calledMethod": "getStatusDocuments",
  "methodProperties": { "Documents": [{ "DocumentNumber": "<ттн>" }] }
}
```

**Логіка існування:**
- `success === true` і `data[0]` присутній
- **Не існує**, якщо `data[0].StatusCode === "0000"` (накладну не знайдено)
- **Існує** — будь-який інший `StatusCode`

**Обробити:** мережеві помилки, порожній `data`, непорожній `errors[]`, rate limiting — усе з дружніми повідомленнями. Винести в `lib/novaposhta.ts` з типізованими request/response.

Примітка: метод `getStatusDocuments` працює і без ключа, але поле ключа лишаємо за вимогою ТЗ і додаємо в запит, якщо він заданий (дає розширені дані).

## Друк (критична частина) — WYSIWYG через системний діалог
1. Code 128 генерується через **bwip-js** у **PNG data URL** (`scale: 4` для чіткості на 203 dpi, `includetext: true`, `textxalign: 'center'` — номер друкується під кодом).
2. **Та сама** картинка йде і в превʼю (`<img>` у React), і в HTML для друку → превʼю дорівнює друку.
3. HTML для друку: `@page { size: {W}mm {H}mm; margin: 0 }` + контейнер рівно під розмір етикетки, картинка по центру.
4. main: прихований `BrowserWindow` → `loadURL('data:text/html,...')` → дочекатись відмалювання → `webContents.print({ silent: false, margins: { marginType: 'none' }, pageSize: { width: W*1000, height: H*1000 } /* мікрони */ }, cb)`.
5. `silent: false` → відкривається **системний діалог** (вимога ТЗ). Не реалізовувати тихий друк.

## Відомі підводні камені — обробити проактивно
- **pageSize валідація Chromium:** мінімум 353 мікрони по кожній стороні; невалідний розмір кидає помилку — валідувати перед друком.
- **Hidden window:** друкувати лише після завантаження й відмалювання (`await loadURL` + невелика затримка або `did-finish-load`), інакше етикетка вийде порожня.
- **Ненадійний callback `webContents.print`:** при скасуванні діалогу `success/reason` можуть не прийти стабільно. Не блокувати критичну логіку на результаті, **не** позначати накладну «надрукованою» автоматично, додати запобіжник, щоб приховане вікно не зависало.
- **macOS:** драйвер XP-365B капризний (працює через CUPS). Перевіряти доступні принтери через `webContents.getPrintersAsync()`; якщо принтера нема — зрозуміле повідомлення. Raw TSPL/ESC-POS у цій ітерації **не** реалізовувати.
- **bwip-js import:** шлях може відрізнятись між версіями (`'bwip-js'` vs `'bwip-js/browser'`) — підібрати правильний для browser-бандла renderer'а.

## Definition of Done
- [ ] Стартує на Windows і macOS (`electron-vite dev`)
- [ ] Валідна ТТН → «знайдено» + превʼю; невалідна → «не знайдено», друк заблоковано
- [ ] Без ключа → підказка про Налаштування
- [ ] Ключ зберігається зашифровано через `safeStorage`, не лежить у plaintext/localStorage
- [ ] «Друкувати» відкриває системний діалог із коректним розміром сторінки; друкується саме те, що в превʼю
- [ ] `contextIsolation` / `nodeIntegration` / `sandbox` налаштовані безпечно
- [ ] `build:win` і `build:mac` виробляють інсталятори
- [ ] TypeScript strict без помилок; обробка помилок на всіх async-шляхах
